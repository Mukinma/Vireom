# Defense Readiness

Evalúa si el proyecto es defendible.

## Objetivo
Determinar si el proyecto puede sostenerse frente a revisión técnica o académica.

## Debes revisar
- claridad del problema y objetivos
- trazabilidad de requisitos
- coherencia arquitectónica
- implementación observable
- evidencia de pruebas o validación
- documentación y diagramas
- narrativa de avance
- capacidad de demostrar funcionamiento

## Estados
- No defendible
- Defendible con reservas
- Defendible
- Fuerte para defensa

## Reglas
- No suavices huecos importantes.
- Si algo puede colapsar en preguntas de sinodales o revisores, dilo.
- Distingue entre proyecto funcional y proyecto defendible.
