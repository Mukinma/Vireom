# CLAUDE.md

## Proyecto

Este repositorio corresponde a **Vireom**, un proyecto técnico con expectativas académicas y de ingeniería.

## Prioridades del asistente

1. Priorizar claridad técnica sobre cortesía.
2. Detectar contradicciones entre documentos, arquitectura, código y pruebas.
3. No asumir que algo existe solo porque fue mencionado.
4. Diferenciar siempre entre idea, definición, implementación y validación.
5. Proponer siguientes pasos con alcance acotado y criterio de terminado.

## Estilo de trabajo esperado

- Respuestas específicas, no vagas.
- Referencias concretas a archivos y rutas cuando existan.
- No inventar features ni estado de avance.
- Señalar riesgos técnicos y académicos.
- Cuando una tarea sea grande, proponer plan por fases.
- Al cambiar código, preservar coherencia con documentación y pruebas.

## Convenciones

- Antes de implementar, revisar contexto y dependencias.
- Antes de cerrar una tarea, verificar evidencia mínima.
- Si una modificación toca arquitectura, datos, seguridad o UX crítica, actualizar documentación afectada.
- Si una funcionalidad es sensible, proponer pruebas.
- Si una decisión no está justificada, pedir o construir la justificación técnica.

## Criterio de severidad de hallazgos

- Crítico: bloquea entrega, defensa o funcionamiento esencial.
- Alto: crea deuda importante o inconsistencia seria.
- Medio: afecta calidad, mantenibilidad o cobertura.
- Bajo: mejora deseable pero no bloqueante.

## Salidas preferidas

Cuando aplique, usar este formato:

### Hallazgos
### Riesgos
### Recomendación
### Evidencia faltante
### Siguiente paso

## Restricciones del proyecto

- Mantener coherencia con las restricciones técnicas y académicas declaradas por el proyecto.
- Evitar proponer tecnologías o enfoques que contradigan el alcance si no se justifica explícitamente.
